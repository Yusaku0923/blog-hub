# `blog-site-template` Repo Creation (M3) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `Yusaku0923/blog-site-template` GitHub Template repository so any future sister site can be spun up with `gh repo create my-new-blog --template Yusaku0923/blog-site-template`.

**Architecture:** Single Nuxt 3 + `@nuxt/content` v3 + Cloudflare Pages site that **extends the `blog-base` Nuxt Layer** (SHA-pinned via git URL). All site-specific files (Header / Footer / Hero / static pages / theme values) ship as **placeholders marked `TEMPLATE:`** and are find-and-replaced at derivation time. Decap CMS admin lives at `public/admin/` and is deployed at `<site>/admin/` by Cloudflare Pages.

**Tech Stack:** pnpm 9, Node 20, Nuxt 3.14+, `@nuxt/content` v3, Decap CMS 3 (admin via `<script>` from unpkg), Tailwind v3, TypeScript strict, GitHub Actions for CI.

**Source spec:** `docs/superpowers/specs/2026-05-07-blog-site-template-design.md`

**Working directory (new):** `/Users/yusaku/projects/blog-site-template/`

**TDD scope:** None of this is business logic. Verification is `pnpm install` + `pnpm typecheck` + `pnpm build` + manual `pnpm dev` visual check + `pnpm template:check` exits 0 in derived sites. No unit tests needed — the same TDD-skip rationale from the M1+M2 plan applies.

- [ ] **Step 3.3: Create `content.config.ts`**

```ts
import { defineCollection, defineContentConfig, z } from '@nuxt/content'

// NOTE: schema kernel duplication.
// blog-base ships a canonical baseSchema at packages/blog-base/content/schema.ts.
// content.config.ts is loaded by c12 before Nuxt aliases / module resolution
// activate, so importing the kernel from the layer fails. Inlining keeps this
// file static-importable. Drift risk is small — kernel rarely changes.
// Keep in sync with: packages/blog-base/content/schema.ts
export default defineContentConfig({
  collections: {
    content: defineCollection({
      type: 'page',
      source: '**/*.md',
      schema: z.object({
        title:       z.string().min(10).max(40),
        description: z.string().min(40).max(140),
        tags:        z.array(z.string()).default([]),
        date:        z.coerce.date(),
        created:     z.coerce.date(),
        listed:      z.boolean().default(true),
        toc:         z.boolean().default(true),
        draft:       z.boolean().default(false),
        pageType:    z.enum(['article', 'category']).default('article'),
        heroImage:   z.string().optional(),
        // TEMPLATE: replace with z.enum([...]) once your categories are defined.
        category:    z.string(),
      }),
    }),
  },
})
```
